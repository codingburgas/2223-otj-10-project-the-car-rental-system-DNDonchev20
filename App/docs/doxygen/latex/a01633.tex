\doxysubsection{Detailed Description}
Specialized \doxylink{a01661}{Sound\+Recorder} which stores the captured audio data into a sound buffer. 

\doxylink{a01633}{sf\+::\+Sound\+Buffer\+Recorder} allows to access a recorded sound through a \doxylink{a01629}{sf\+::\+Sound\+Buffer}, so that it can be played, saved to a file, etc.

It has the same simple interface as its base class (\doxylink{a01661_a715f0fd2f228c83d79aaedca562ae51f}{start()}, \doxylink{a01661_a8d9c8346aa9aa409cfed4a1101159c4c}{stop()}) and adds a function to retrieve the recorded sound buffer (\doxylink{a01633_a1befea2bfa3959ff6fabbf7e33cbc864}{get\+Buffer()}).

As usual, don\textquotesingle{}t forget to call the \doxylink{a01661_aab2bd0fee9e48d6cfd449b1cb078ce5a}{is\+Available()} function before using this class (see \doxylink{a01661}{sf\+::\+Sound\+Recorder} for more details about this).

Set the processing interval. 

The processing interval controls the period between calls to the on\+Process\+Samples function. You may want to use a small interval if you want to process the recorded data in real time, for example.

Note\+: this is only a hint, the actual period may vary. So don\textquotesingle{}t rely on this parameter to implement precise timing.

The default processing interval is 100 ms.

Start the capture. 

The {\itshape sample\+Rate} parameter defines the number of audio samples captured per second. The higher, the better the quality (for example, 44100 samples/sec is CD quality). This function uses its own thread so that it doesn\textquotesingle{}t block the rest of the program while the capture runs. Please note that only one capture can happen at the same time. You can select which capture device will be used, by passing the name to the \doxylink{a01661_a8eb3e473292c16e874322815836d3cd3}{set\+Device()} method. If none was selected before, the default capture device will be used. You can get a list of the names of all available capture devices by calling \doxylink{a01661_a2a0a831148dcf3d979de42029ec0c280}{get\+Available\+Devices()}.
