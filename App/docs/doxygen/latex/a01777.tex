\doxysubsection{Detailed Description}
\doxylink{a01693}{Drawable} representation of a texture, with its own transformations, color, etc. 

\doxylink{a01777}{sf\+::\+Sprite} is a drawable class that allows to easily display a texture (or a part of it) on a render target.

It inherits all the functions from \doxylink{a01793}{sf\+::\+Transformable}\+: position, rotation, scale, origin. It also adds sprite-\/specific properties such as the texture to use, the part of it to display, and some convenience functions to change the overall color of the sprite, or to get its bounding rectangle.

\doxylink{a01777}{sf\+::\+Sprite} works in combination with the \doxylink{a01785}{sf\+::\+Texture} class, which loads and provides the pixel data of a given texture.

The separation of \doxylink{a01777}{sf\+::\+Sprite} and \doxylink{a01785}{sf\+::\+Texture} allows more flexibility and better performances\+: indeed a \doxylink{a01785}{sf\+::\+Texture} is a heavy resource, and any operation on it is slow (often too slow for real-\/time applications). On the other side, a \doxylink{a01777}{sf\+::\+Sprite} is a lightweight object which can use the pixel data of a \doxylink{a01785}{sf\+::\+Texture} and draw it with its own transformation/color/blending attributes.

It is important to note that the \doxylink{a01777}{sf\+::\+Sprite} instance doesn\textquotesingle{}t copy the texture that it uses, it only keeps a reference to it. Thus, a \doxylink{a01785}{sf\+::\+Texture} must not be destroyed while it is used by a \doxylink{a01777}{sf\+::\+Sprite} (i.\+e. never write a function that uses a local \doxylink{a01785}{sf\+::\+Texture} instance for creating a sprite).

See also the note on coordinates and undistorted rendering in \doxylink{a01793}{sf\+::\+Transformable}.

The {\itshape texture} argument refers to a texture that must exist as long as the sprite uses it.
